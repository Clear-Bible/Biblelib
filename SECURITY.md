# Security Policy

## Supported Versions

We support the latest code on the `main` branch.

Older commits, branches, and unreleased local modifications may not receive security fixes.

## Reporting a Vulnerability

Please do **not** open a public GitHub issue for suspected security vulnerabilities.

Instead, report vulnerabilities privately to the maintainers at:

- sean.boisen@biblica.com

Please include:

- a clear description of the issue
- affected version, tag, or commit SHA
- steps to reproduce
- proof-of-concept code or screenshots, if available
- potential impact
- any suggested remediation

## What to Expect

We will try to:

- acknowledge receipt within 5 business days
- assess severity and impact
- provide a remediation plan or timeline when possible
- coordinate disclosure after a fix is available

## Scope

This policy applies to:

- the source code in this repository
- release artifacts published from this repository
- CI/CD workflows and repository configuration associated with this project

## Out of Scope

The following are generally out of scope unless they create a clear security impact:

- spelling, formatting, or documentation errors
- purely theoretical issues without a plausible exploitation path
- vulnerability reports for unsupported forks or modified downstream deployments

## Disclosure

Please allow time for investigation and remediation before public disclosure.
We appreciate responsible disclosure and will work in good faith with reporters.
